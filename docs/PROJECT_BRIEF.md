# Project Brief

AgentLighthouse helps companies answer whether AI coding agents can correctly understand and use their product, repository, API, documentation, MCP server, and developer workflows.

## Target Users

- Developer-tool startups.
- API providers.
- SDK maintainers.
- Documentation teams.
- Engineering platform teams.
- Companies adopting AI coding agents across internal repositories.

## Pain Points

- Agents fail because setup commands, architecture rules, and task workflows are implicit.
- Docs are written for humans but not discoverable or actionable for machines.
- API schemas exist without examples or meaningful descriptions.
- Agent instruction files are missing, stale, or inconsistent across tools.
- Teams lack a repeatable readiness signal before adding agents to CI or developer workflows.

## Why This Complements Agents

AgentLighthouse does not execute coding tasks as the primary product. It improves the environment that Codex, Claude Code, Cursor, Copilot, Gemini, and future agents read. Better context makes every agent more reliable.
