# v0.1.0-alpha.1 Release Report

Status: pending recovery release.

## Purpose

`v0.1.0-alpha.1` is an incident recovery release for npm installability. It replaces the
broken CLI package metadata published in `0.1.0-alpha.0`.

## Planned Package Versions

- `@agentlighthouse/core@0.1.0-alpha.1`
- `@agentlighthouse/cli@0.1.0-alpha.1`

## Required Verification Before Announcement

- Packed package metadata contains no `workspace:` dependency protocols.
- A clean npm consumer can install the packed CLI tarball.
- A clean npm consumer can install `@agentlighthouse/cli@alpha` from the registry after
  publish.
- `npx agentlighthouse version` reports `0.1.0-alpha.1`.
- `npx @agentlighthouse/cli@alpha version` reports `0.1.0-alpha.1`.
- `alpha` dist-tags point to `0.1.0-alpha.1`.
- `latest`, if present, does not point to the broken CLI `0.1.0-alpha.0`.

## Rollback Notes

- npm versions cannot be overwritten.
- Publish `0.1.0-alpha.2` for any follow-up fix.
- npm dist-tags can be moved if necessary.
- The first stable release should move `latest` to a stable version.
- GitHub release notes can be edited.
- Do not rewrite pushed public tags without explicit approval.
