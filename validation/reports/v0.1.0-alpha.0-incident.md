# v0.1.0-alpha.0 npm Incident

## Summary

`@agentlighthouse/cli@0.1.0-alpha.0` was published to npm with an invalid dependency
specifier:

```json
{
  "@agentlighthouse/core": "workspace:^"
}
```

Clean npm consumers cannot install the CLI package because npm rejects workspace protocol
dependencies outside a workspace.

## Impact

- `@agentlighthouse/core@0.1.0-alpha.0` was published and is installable.
- `@agentlighthouse/cli@0.1.0-alpha.0` was published but is not installable by npm
  consumers.
- `npm install @agentlighthouse/cli@alpha` failed with
  `EUNSUPPORTEDPROTOCOL Unsupported URL Type "workspace:"`.
- The `v0.1.0-alpha.0` git tag and GitHub prerelease remain valid historical artifacts and
  must not be moved.

## Detection

The issue was found during post-release npm consumer verification, after publishing but before
announcing the release.

## Root Cause

The release smoke tests installed packed tarballs through pnpm with workspace overrides. That
masked the fact that the packed CLI package metadata still contained a workspace protocol
dependency. The release checks did not inspect the packed `package/package.json` dependency
metadata for `workspace:` values.

## Recovery Plan

- Burn `0.1.0-alpha.0`; do not overwrite or unpublish it.
- Publish `@agentlighthouse/core@0.1.0-alpha.1`.
- Publish `@agentlighthouse/cli@0.1.0-alpha.1`.
- Use an exact CLI dependency on `@agentlighthouse/core@0.1.0-alpha.1`.
- Move the `alpha` dist-tag to `0.1.0-alpha.1`.
- Ensure `latest`, if present, does not point to the broken CLI version.
- Deprecate `@agentlighthouse/cli@0.1.0-alpha.0` with a warning that users should install
  `0.1.0-alpha.1` or later.

## Prevention Added

- Packed package metadata is inspected for `workspace:` dependency protocols.
- `release:smoke` now installs packed tarballs into a clean npm consumer project.
- Fresh-clone rehearsal uses npm for the packed CLI consumer install path.
